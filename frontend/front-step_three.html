Шаг №3 Cascading Style Sheets

Основы CSS

Структура правила CSS

Селектор - идентифицирует элементы, на которые оказывается воздействие.

Объявление - содержит инструкции по визуализации и состоит из свойства и его значения.

Разделы стилевого правила:селектор { property:value; }

Комментарии - в таблицах стилей применяется следующий синтаксис /*здесь находится комментарий*/.

Способы подключения таблицы стилей

  • Внешние таблицы стилей - отдельный текстовый документ, содержащий несколько правил стилей. Представляет собой файл с расширением css. Файл связывается с помощью элемента link или импортируется с помощью правила @import.

  • Вложенные таблицы стилей - помещаются в документ с помощью элемента < style >, который помещается в заголовке документа.

  • Встроенные таблицы стилей - свойства стиля применяются к одному элементу с помощью атрибута style, несколько стилей разделяются точкой с запятой. Лучше не использовать.

Важные понятия

  • Наследование - передача свойств по иерархии структуры html-документа. Любое примененное к конкретному элементу свойство, переопределяет унаследованные этим элементом свойства.

  • Каскад - применение нескольких таблиц стилей к одному документу. Управление стилем элементов, если на странице возникает конфликт нескольких источников информации о стиле. Переопределение стиля осуществляется на основании веса правила стиля. Вес стиля формируется на основе приоритета источника для правила стиля специфичности селектора и порядка следования правил.

  • Приоритет источника - имеет следующий порядок от более приоритетного к менее:
    1) любое правило стиля, помеченное как < !important > читателем (пользователем).Пример применения: < p{color: blue !important;} >;
    2) таблицы стилей, составленные авторами веб-страницы;
    3) таблицы стилей, сформированные читателем (пользователем);
    4) заданные по умолчанию правила стилей браузера ("таблица стилей пользовательского агента").

  • Специфичность - некоторые селекторы имеют больший "вес" и имеют приоритет над другими селекторами:ЗАПОЛНИТЬ ПОЗДНЕЕ

  • Правило следования - для правил имеющих равные "веса"" и одинаковый приоритет источника, решающим является какое из правил появилось последним.

  • Блочная модель (box model) - браузеры представляют каждый элемент на странице как заключенные в прямоугольные блоки.ЗАПОЛНИТЬ ПОЗДНЕЕ

Единицы измерения

Абсолютные единицы - имеют предопределенные значения (для веб-страниц, отображаемых на экране подходит только пиксель (px).

  • px - пиксели 1/96 дюйма.

  • in - дюймы.

  • mm (cm) - миллиметры (сантиметры).

  • q - 1/4 миллиметра.

  • pt - пункты (1/2 дюйма), применяется в полиграфическом дизайне.

  • pc - пики (1 пика=1/72 дюйма), применяется в полиграфическом дизайне.

Относительные единицы - основаны на сравнении с размером другого элемента (размер текста или размер родительского элемента).

  • em - равна текущему размеру шрифта (ширина буквы "M"), расстояние между базовыми линиями (когда шрифт без дополнительного пространства между строками "лидинг").

  • ex - "x-высота" приблизительно равна высоте строчной буквы "х" в шрифте.

  • rem - root em, равный размеру корневого элемента em HTML.

  • ch - нулевая ширина, равная ширине нуля (0) в текущем шрифте и размере.

  • vw - равная 1/100 ширины текущего окна просмотра браузера.

  • vh - равная 1/100 высоты текущего окна просмотра браузера.

  • vmin - минимальная единица просмотра, равная значению vw или vh, в зависимости от того, что из них меньше.

  • vmax - максимальная единица просмотра, равная значению vw или vh, в зависимости от того, что из них больше.

  • % - процент относительно другого значения.

СЕЛЕКТОРЫ

Виды селекторов

  • Селекторы типа элемента - свойства для каждого правила применяются к каждому элементу, например: h1 или p , p{color:navy;}. Ссылка на пример.

  • Сгруппированные селекторы - выбор элемента на основе группы селекторов, выбранные элементы разделяются запятой - p, ul, li, td {color:navy;}

  • Селекторы ID, предназначается для элемента, который имеет уникальное имя - p#id {color:navy;}.

  • Селекторы класса, предназначается для элемента, который принадлежит к определенному классу - p.class {color:navy;}

  • Селекторы потомков (child selectors) - p ul {color:navy;}

  • Дочерний селектор, указывают только на прямых потомков элемента, между которыми не может быть других иерархических уровней - p>ul {color:navy;}

  • Селекторы ближайших братьев и сестер (next-sibling), предназначается для элемента, который идет непосредственно после другого элемента с тем же родителем - p+ul {color:navy;}.

  • Селекторы последующих братьев и сестер (following-sibling), предназначается для элемента, который идет после другого элемента с тем же родителем - p~ul {color:navy;}.

  • Универсальные селекторы, соответствует любому элементу - p * {color:navy;}

  • Селекторы псевдоклассов,отслеживание состояний элементов, обозночаются двоеточием (:)- li:first-child.
    Основные селекторы псевдоклассов (указаны в обязательном порядке для элементов):

    • :link - применяет стиль к неактивным ссылкам. Является псевдоклассом для ссылок.

    • :visited - применяет стиль, на которых выполнен щелчок ЛКМ. Могут использовать только следующие свойства color, background-color, border-color, outline-color. Является псевдоклассом для ссылок.

    • :target - идентификатор фрагмента. Является псевдоклассом для ссылок (местоположения).

    • :focus - применяется, если элемент выбран и готов к вводу.Является псевдоклассом действия.

    • :hover - применяется, если указатель мыши находится над элементом.Является псевдоклассом действия.

    • :active - применяется, когда элемент участвует в процессе нажатия или касания. Является псевдоклассом действия.

    • :root, :empty, :first-child, :last-child, :only-child, :first-of-type, last-of-type, :only-of-type, :nth-child(), :nth-last-child(), :nth-of-type(), :nth-lasat-of-type - применяются для выбора элемента в зависимости от его места в структуре документа.Являются структурными псевдоклассами.

    • :enabled, :disabled, :checked - применяются к состояниям, типичным для ввода формы.Являются псевдоклассами ввода.

    • :enabled, :disabled, :checked - применяются к состояниям, типичным для ввода формы.Являются псевдоклассами ввода.

    • :lang() - лингвистический псевдокласс.

    • :not() - логический псевдокласс.

  • Селекторы псевдоэлементов, вставляют в структуру документа вымышленные элементы, обозначаются двойным двоеточием (::) - p::first-line{letter-spacing:9px;}. Селектор псевдоэлемента указывается после целевого элемента без пробела, с обязательным применением свойства content,
    Основные селекторы псевдоэлементов:

    • ::first-line- задает правило стиля для первой строки указанного элемента. Относится только к: color, font properties, background properties, word-spacing, letter-spacing, text-decoration, vertical-align, text-transform, line-height.

    • ::first-letter- задает правило стиля для первой буквы указанного элемента. Относится только к: color, font properties, background properties, letter-spacing, word-spacing, text-decoration, vertical-align (если нет float), text-transform, padding properties, margin properties, border properties, float, line-height.

    • ::before и ::after - добавление сгенерированного контента до или после выбранного элемента - p.warning::before{content:url(image.png);}.

  • Селекторы атрибутов,выбирают элементы на основе имен или значений атрибутов.
    Основные селекторы атрибутов:

    • element[атрибут]- простой селектор атрибута выбирает целевые элементы с определенным атрибутом независимо от его назначения.

    • element[атрибут="точное значение"]- селектор точного значения атрибута выбирает целевые элементы с определенным значением атрибута.

    • element[атрибут~="значение"]- селектор частичного значения атрибута, позволяет указать одну часть из значения атрибута.

    • element[атрибут|="значение"]- селектор значения атрибута, разделенного дефисами, выбирает разделенные дефисами значения атрибута.

    • element[атрибут^="первая часть значения"]- селектор значения атрибута начальной подстроки.

    • element[атрибут$="последняя часть значения"]- селектор значения атрибута последней подстроки.

    • element[атрибут*="любая часть значения"]- селектор значения атрибута любой подстроки.

Специфичность

Порядок и соотношение весов селекторов:

  • встроенные стили более специфичны чем...(1000)

  • селекторы id более специфичны чем...(100)

  • селекторы класса более специфичны чем... (10)

  • индивидуальные селекторы элементов (1).

Форматирование текста

Свойства шрифтов

Ключевые слова для свойств CSS:

  • initial

    Является широко применяемым соглашением, например стандарт определяет значения атрибутов id, class, rel для определения людей, организаций, событий, продуктов и др.

  • Microdata

    Является HTML-стандартом WHATWG (Web Hypertext Application Technology Working Group, Рабочая группа по технологиям веб-гипертекста), который применяет специфические атрибуты: item-scope, itemtype, itemprop, itemid, itemref. Данный стандарт в 2013 году исключен из HTML5

  • RDFa

    Является HTML-стандартом (Resource Description Framework in Attributes, Фреймворк описания ресурсов в атрибутах), который применяет специфические атрибуты: vocab, typeof, property, resourse, prefix.

  • RDFa Lite

    Является упрощенной версией RDFa.

  • JSON-LD

    Другой подход в части размещения типов объектов и их свойств в удаленном из разметки сценарии (JavaScript Object Notation to serialize Linked Data, запись объекта JavaScript, применяемая для сериализации связанных данных).

  • Общие семейства шрифтов: - браузер выбирает доступный шрифт из стилистической категории:

    • serif - шрифты с засечками.

    • sans-serif - шрифт без засечек.

    • monospace - моноширный шрифт, все символы занимают в строке одинаковое пространство.

    • cursive - курсивный шрифт.

    • fantasy - декоративные шрифты (для заголовков).

  • Требования к синтаксису:

    • все названия шрифтов, за исключением общих семейств шрифтов, должны быть написаны с заглавной буквы;

    • используйте запятые для разделения нескольких названий шрифтов;

    • имена шрифтов, содержащие пробелы, должны заключаться в кавычки.

  • font-family - выбор семейства шрифтов. Относится к CSS2.1.

    • Значения: один или несколько шрифтов или названия общих шрифтов, разделенные запятыми.

    • По умолчанию: зависит от браузера.

    • Применение: ко всем элементам.

    • Наследование: да.

  • font-size - указание размера шрифтов. Относится к CSS2.1. Задать размер можно с помощью одной из единиц длины CSS (em, rem), с помощью процентов, с помощью абсолютных ключевых слов (small).

    • Значения: единица длины|процентное соотношение|xx-small|x-small|small|medium|large|x-large|xx-large|smaller|larger.

    • По умолчанию: medium.

    • Применение: ко всем элементам.

    • Наследование: да.

    В CSS3 добавлено свойство font-size-ajdjust, которое позволяет отрегулировать высоту резервного шрифта по x-высоте отобранного шрифта.

  • font-weight - "жирность" шрифтов. Относится к CSS2.1.

    • Значения: normal|bold|bolder|lighter|100|200|300|400|500|600|700|800|900.

    • По умолчанию: normal.

    • Применение: ко всем элементам.

    • Наследование: да.

  • font-style - стиль текста, определяет вертикальны или наклонны очертания букв (курсив). Относится к CSS2.1.

    • Значения: normal|italic|oblique.

    • По умолчанию: normal.

    • Применение: ко всем элементам.

    • Наследование: да.

  • font-variant - капитель (small-caps), дизайн в котором строчные знаки выглядят как уменьшенные заглавные. Относится к CSS2.1.

    • Значения: normal|small-caps.

    • По умолчанию: normal.

    • Применение: ко всем элементам.

    • Наследование: да.

    font-variant предоставляет в CSS3 доступ к шрифтам со специальными символами - глифами:

    • font-variant-ligatures - глиф, объединяющий в однин символ два или более символов (список значений можно найти на www.w3.org)

    • font-variant-caps - позволяет из набора символов шрифта выбирать глифы капители. Используются значения: all-smallcaps, применение капители вместо строчных и прописных букв;unicase, применение капители только к символам в верхнем регистре; titling-caps,применение капители для заголовков.

    • font-variant-position - из набора выбирает глифы верхнего индекса (super) или нижнего (sub), если они доступны.

    • font-variant-numeric - позволяет выбирать различные стили числовых символов.

    • font-variant-alternates - предлагают для одного символа несколько вариантов написания.

    • font-variant-east-asian - позволяет выбирать азиатские глифы.

  • font-stretch - сжатый и разреженный шрифт. Относится к CSS2.1.

    • Значения: normal|ultra-condenced|extra-condenced|condenced|semi-condenced|expanded|extra-expanded|ultra-expanded.

    • По умолчанию: normal.

    • Применение: ко всем элементам.

    • Наследование: да.

  • font - сокращенное свойство, которое объединяет в одно правило все связанные со шрифтом свойства. Относится к CSS2.1.

    • Значения: font-style font-weight font-variant font-stretch font-size/line-height font-family|caption|icon|menu|message-box|small-caption|status-bar.

    • По умолчанию: для каждого свойства в правиле зависит от значения, заданного по умолчанию.

    • Применение: ко всем элементам.

    • Наследование: да.

    В CSS3 добавлены свойства:
    font-kerning, которое позволяет применять метаданные о кернинге шрифта (normal), отключать (none), или оставлять на усмотрение браузера (auto);
    font-feature-settings, которое позволяет управлять расширенными типографскими функциями в шрифтах OpenType.

  • color - изменение цвета. Свойство применяется для изменения цвета элемента на переднем плане (не фона), передний план включает текст и его границы.

    • Значения: значение цвета (название или числовое значение).

    • По умолчанию: зависит от браузера и предпочтений пользователя.

    • Применение: ко всем элементам.

    • Наследование: да.

    В CSS2 определены 17 стандартных названий цветов: black,lime,silver,olive,gray,yellow,white,navy,maroon,blue,red,teal,purple,aqua,fuchsia,orange,green;
    в CSS3 возможно задавать значения по названию 140 цветов.

Настройки текстовых строк

  • Свойство line-height - высота строки, определяет минимальное расстояние между базовыми линиями строк текста

    • Значения: number|lenght measurement|percentage|normal.

    • По умолчанию: normal.

    • Применение: ко всем элементам.

    • Наследование: да.

  • Свойство text-ident -отступ первой строки текста на указанную величину.

    • Значения: величина|процентное соотношение.

    • По умолчанию: 0.

    • Применение: к блочным контейнерам.

    • Наследование: да.

  • Свойство text-align - горизонтальное выравнивание текста.

    • Значения: left|right|center|justify|start|end.

    • По умолчанию: start.

    • Применение: к блочным контейнерам.

    • Наследование: да.

    В CSS3 добавлены значения start и end, которые определяют сторону строки, по которой проводится выравнивание текста (для языков, написание которых осуществляется справа налева или по вертикали).

  • Свойство text-decoration - украшательства текста.

    • Значения: none|underline(подчеркивание)|overline(надчеркивание)|line-through (перечеркивание)|blink (мерцание).

    • По умолчанию: none.

    • Применение: ко всем элементам.

    • Наследование: нет, но поскольку строки проходят через дочерние элементы, они могут иметь такой вид, как будто также изменены.

    В CSS3 добавлены значения text-decoration-line, text-decoration-color, text-decoration-style, text-decoration-skip, text-underline-position.

  • Свойство text-transform - изменение вида заглавных букв.

    • Значения: none|capitalize(первая буква заглавная)|lowercase (все буквы строчные)|uppercase (все буквы заглавные)|full-width (выбирает из шрифта полноразмерную версию символа).

    • По умолчанию: none.

    • Применение: ко всем элементам.

    • Наследование: да.

  • Свойство letter-spacing - вставка пробелов между буквами.

    • Значения: величина|normal.

    • По умолчанию: normal.

    • Применение: ко всем элементам.

    • Наследование: да.

  • Свойство word-spacing - вставка пробелов между словами.

    • Значения: величина|normal.

    • По умолчанию: normal.

    • Применение: ко всем элементам.

    • Наследование: да.

  • Свойство text-shadow - добавление к тексту "тени".

    • Значения: 'смещение по горизонтали' 'смещение по вертикали' 'радиус размытия' 'цвет'|none.

    • По умолчанию: none.

Улучшение доступности ARIA

Улучшение доступности документа с помощью ARIA(Accessible Rich Internet Application, Доступные многофункциональные интернет-приложения).

Стандартизированный набор атрибутов, использование которых упрощает навигацию по страницам и применение интерактивных функций.Определяет роли, состояния и свойства, которые могут добавлять в разметку и сценарии для поддержки более насыщенной семантической информации Поддерживается Рабочей группой инициативы доступности веб-сайтов (WAI).

Роли описывают функции или цели элемента в контексте документа.Являются подсказкой для ассистивных технологиях. К ролям относятся: alert, button, dialog, slider, menubar, navigation, banner, contentinfo, complementary, main.Пример применения - < ul id="table" role="toolbar" >.

Состояния и свойства обозначаются атрибутами с префиксом aria-, например: aria-disabled, aria-describedby.

Подключение внешней таблицы стилей

  • Применение элемента link - определяет связь между текущим документом и внешним ресурсом, включается в заголовок текущего документа. Можно включить несколько элементов link, последняя ссылка переопределит предыдущие настройки.

    Атирбуты:

    • rel="stylesheet" - определяет отношение связанного документа к текущему документу (для ссылки на таблицу стилей всегда "stylesheet").

    • href="url-ссылка" - указывает местоположение таблицы стилей(*.css).

  • Применение at-правила @import - позволяет добавить таблицу стилей либо во внешнем документе таблицы стилей, либо в элементе style. В таблице стилей at-правило приводится в начале таблицы стилей перед любыми селекторами.

  • Применение at-правила @import с медиазапросами - позволяет ограничить импорт таблицы стилей определенными типами мультимедиа (например, экран, печать) или средами просмотра (ориентация, размер экрана), медиазапросы записываются через запятую после правила @import, пример @import url(table.css) screen and (max-width:320px).

  • Применение модульных таблиц стилей с использованием правила @import - для использование заранее разработанных стилевых решений, рекомендуется их скомпилировать с помощью специальных инструментов - CSS-препроцессоров, например LESS и Sass.

Плавающие элементы и позиционирование

  • Режим плавания float: - позволяет смещать элемент влево или вправо, а также вызывает обтекание его окружающим текстом.

    • Значения: left|right|none.

    • По умолчанию: auto.

    • Применение: ко всем элементам.

    • Наследование: нет.

    Правила применения "плавающих" элементов:

    1. "Плавающий" элемент удаляется из потока, но влияет на окружающий его контент.
    2. "Плавающий" элемент размещается в области контента элемента, вокруг него могут задаваться поля.
    3. Для "плавающих" элементов необходимо обязательно задавать ширину (свойство width), за исключением img.
    4. "Плавающие" текстовые элементы ведут себя как блочные, при этом верхние и нижние поля не коллапсируют.
    5. "Плавающие" элементы не перемещаются выше места своего размещения в нормальном потоке.

  • Очистка "плавающих" элементов clear: - позволяет отключать обтекание текстов и возвращаться к обычному потоку (для элемента следующего за "плавающим").

    • Значения: left|right|both|none.

    • По умолчанию: none.

    • Применение: только к элементам блочного уровня.

    • Наследование: нет.

  • Свободное обтекание текстом CSS-форм shape-outside: - позволяет изменять форму обтекающего текста на круглую, элипсовидную, многоугольную и др.

    • Значения: none|circle(радиус)|ellipse(rx ry at x y)|polygon(x1 y1 x2 y2...)|url()|[margin-box|padding-box|content-box].

    • По умолчанию: none.

    • Применение: к "плавающим" элементам.

    • Наследование: нет.

  • Свойство shape-margin: - позволяет добавлять свободное пространство между плавающим элементом и текстом.

    • Значения: длина|процентное соотношение.

    • По умолчанию: 0.

    • Применение: к "плавающим" элементам.

    • Наследование: нет.

    Для изображений имеющих несколько уровней прозрачности возможно использовать свойство shape-image-threshold, которое определяет порог прозрачности (значения от 0 до 1).

  • Позиционирование position: - позволяет позиционировать элементы, относительно того места, где они появляются в потоке, или могут быть удалены и размещены в определенном месте на странице.

    • Значения: static (размещение элемента соответствует нормальному потоку)|relative (перемещение элемента относительно его места в потоке)|absolute (элемент удаляется из потока и позиционируется относительно области просмотра или содержащего его элемента)|fixed (фиксация элемента в одной позиции окна просмотра даже при прокрутке документа)|sticky (элемент позиционируется относительно, пока не будет прокручен до указанного положения, после чего становится фиксированным).

    • По умолчанию: static.

    • Применение: ко всем элементам.

    • Наследование: нет.

    Значение relative без указания позиции применяется для организации контекста позиционирования для блока, содержащего элемент, который необходимо абсолютно позиционировать.

  • Указание позиции top, right,bottom,left: - указание фактического положения элемента.

    • Значения: длина|процентное соотношение|auto.

    • По умолчанию: auto.

    • Применение: к позиционируемым элементам.

    • Наследование: нет.

  • Свойство z-index: - изменение порядка наложения элементов при перекрывающем позиционировании.

    • Значения: число|auto.

    • По умолчанию: auto.

    • Применение: к позиционируемым элементам.

    • Наследование: нет.

Flexbox

  • Модуль CSS Flexible Layout: - CSS-модуль макета гибкого контейнера (Flexbox) предназначен для размещения компонентов веб-страниц на основе гибкого макета (flex-макета) по одной оси вертикально или горизонтально (ось может складываться несколько раз).

    Кроме указанного модуля для макетирования в CS3 используется многоколоночный макет - модуль Multi-column Layout Module, который имеет инструменты для разбивки текстового контента на несколько столбцов.

  • Настройка flex-контейнера, свойство display:inline-flex (flex): - генерирует линейный уровень flex-контейнера.

  • Задание направления потока flex-direction:

    • Значения: row|column|row-reverse|column-reverse.

    • По умолчанию: row.

    • Применение: к flex-контейнерам.

    • Наследование: нет.

  • Разбивка группы элементов на несколько линий flex-wrap:.

    • Значения: nowrap|wrap|wrap-reverse.

    • По умолчанию: nowrap.

    • Применение: к flex-контейнерам.

    • Наследование: нет.

  • Совмещение свойств flex-direction и flex-wrap flex-flow:.

    • Значения: значение flex-direction|значение flex-wrap.

    • По умолчанию: row nowrap.

    • Применение: к flex-контейнерам.

    • Наследование: нет.

  • Выравнивание и распределение вдоль главной оси justify-content, применяется к контейнеру:.

    • Значения: flex-start|flex-end|center|space-between (первый и последний элементы размещаются по краям, остальные распределяются)|space-around (добавляет пространство слева и справа)

    • По умолчанию: flex-start.

    • Применение: к flex-контейнерам.

    • Наследование: нет.

  • Выравнивание и распределение вдоль поперечной оси align-items, применяется к контейнеру.

    • Значения: flex-start|flex-end|center|baseline (выравнивание базовых линий элементов)|stretch(элементы растягиваются для заполнения поперечной оси)

    • По умолчанию: stretch.

    • Применение: к flex-контейнерам.

    • Наследование: нет.

  • Выравнивание и распределение вдоль поперечной оси align-self.

    • Значения: flex-start|flex-end|center|baseline (выравнивание базовых линий элементов)|stretch(элементы растягиваются для заполнения поперечной оси)

    • По умолчанию: stretch.

    • Применение: к элементам flex-контейнера.

    • Наследование: нет.

  • Выравнивание и распределение нескольких flex-линий (только для wrap и wrap-reverse) align-content, распределяют дополнительное пространство вокруг нескольких линий.

    • Значения: flex-start|flex-end|center|space-around|space-between|stretch

    • По умолчанию: stretch.

    • Применение: к многострочным flex-контейнерам.

    • Наследование: нет.

  • Сокращенное свойство flex , указывает насколько элемент может увеличиваться и уменьшаться, и задает его начальный размер.Предпочтительно не менять свойства flex-grow flex-shrink flex-basis по отдельности

    • Значения: none|'flex-grow flex-shrink flex-basis'

    • По умолчанию: 0 1 auto.

    • Применение: к flex-элементам.

    • Наследование: нет.

    Сокращенные значения: flex:initial - равно flex:0 1 auto (препятствует растягиванию даже при наличии свободного пространства); flex:auto - равно flex: 1 1 auto (элементы полностью гибкие); flex:none - равно flex:0 0 auto (негибкий элемент); flex:целое число - равно flex:целое число 1 0px (элементы пропорциональны своим абсолютным значениям отностительно flex-grow)

  • Свойство растягивающихся элементовflex-grow, определяет каким образом элемент можно растягивать.

    • Значения: число.

    • По умолчанию: 0.

    • Применение: к flex-элементам.

    • Наследование: нет.

  • Свойство сжатия элементовflex-shrink, определяет каким образом элемент можно сжимать.

    • Значения: число.

    • По умолчанию: 0.

    • Применение: к flex-элементам.

    • Наследование: нет.

  • Свойство, определяющее начальный размер элемента, до его выравнивания, растягивания или сжатия,flex-basis.

    • Значения:длина|процентное соотношение|content|auto.

    • По умолчанию: auto.

    • Применение: к flex-элементам.

    • Наследование: нет.

  • Изменение порядка следования flex-элементов order. Инструмент для адаптивного дизайна.

    • Значения:целое число.

    • По умолчанию: 0.

    • Применение: к flex-элементам и абсолютно позиционируемым дочерним элементам flex-контейнеров.

    • Наследование: нет.

CSS Grid

  • Модуль CSS GRID Layout: - Процесс использования модуля CSS GRID Layout включает:.

    1. Применить свойство display для превращения элемента в grid-контейнер - его дочерние элементы автоматически превратятся в grid-элементы.

    2. Настройте для сетки столбцы и строки - задать явно и/или указать как они формируются "на лету".

    3. Присвоить каждому элементу сетки соответствующую область в сетке - если не присвоисть явно, элементы разместятся последовательно.

  • Объявление отображения сетки grid (inline-grid):.

  • Определение дорожек сетки grid-template-rows, grid-template-columns:.

    • Значения: none|список размеров дорожек и (опционно) названий линий.

    • По умолчанию: none.

    • Применение: к grid-контейнерам.

    • Наследование: нет.

    Каждой линии сетки присваивается номер: слева направо, сверх вниз положительные числа, справа налево снизу - отрицательные числа. Кроме этого каждой линии можно присвоить имя следующим образом grid-template-rows:[header-start] 100px [content-start] 400px [footer-start] 100px. Можно использовать несколько имен, указав их в одних квадратных скобках.

    Значения размеров дорожки:

    • Длины (px, em и др.).

    • Процентные значения.

    • Дробные единицу (fr) - позволяет объединять фиксированные и гибкие размеры дорожек, а также задавать пропорции размеров.

    • auto, min-content, max-content - устанавливают размеры дорожек, исходя из контента.

    • minmax(15em, 45em) - позволяет задавать ограничения размеров.

    • fit-content().

    • Функция repeat (количество повторений, шаблон размеров дорожек (20px 1fr)) с опциональными значениями: repeat(auto-fill, 10em - формирует строки каждые 10em, и auto-fit - аналогично, за исключением того, что из макета удаляются дорожки без контента.

  • Назначение имен областям сетки grid-template-areas:.

    • Значения: none|набор имен областей сетки ("header header header" "aside main articale" "footer footer footer"). Если необходимо оставить ячейку без имени - вместо имени ставится точка или многоточие.

    • По умолчанию: none.

    • Применение: к grid-контейнерам.

    • Наследование: нет.

  • Сокращенное свойствоgrid:.

    • Значения: none|информация о строке/информация о столбце ([header-start] "header header header" 100px [content-start] "aside main articale" 400px [footer footer footer] 100px/[aside] 200px [main] 1fr [articale] 200px).

    • По умолчанию: none.

    • Применение: к grid-контейнерам.

    • Наследование: нет.

  • Позиционирование grid-элементов с помощью линийgrid-row-start, grid-row-end, grid-column-start, grid-column-end:.

    • Значения: auto|линия сетки|span количество|span 'имя линии'|число 'имя линии'.

    • По умолчанию: auto.

    • Применение: к grid-элементам.

    • Наследование: нет.

    Сокращенные свойства grid-row, grid-column имеют значения - начальная линия/конечная линия grid-column:1/span 3, объединяют свойства *-start и *-end в одно объявление.

  • Позиционирование области grid-area - указывает на одну из областей, именованных с помощью свойства grid-template-areas.

    • Значения: название области|от 1 до 4 идентификаторов линий.

    • По умолчанию: отдельные свойства.

    • Применение: к grid-элементам.

    • Наследование: нет.

  • Автоматическое указание размера неявным строкам и столбцамgrid-auto-rows, grid-auto-columns.

    • Значения: список размеров дорожек.

    • По умолчанию: auto.

    • Применение: к grid-контейнерам.

    • Наследование: нет.

  • Направление потока элементов и плотность их расположенияgrid-auto-flow.

    • Значения: row или column|dense (опционально,заполняет сетку настолько плотно, насколько это возможно, позволяя элементам размещаться не по порядку, но заполняя своное пространство).

    • По умолчанию: row.

    • Применение: к grid-контейнерам.

    • Наследование: нет.

  • Пространство между дорожками ("желоба")grid-row-gap, grid-column-gap.

    • Значения: длина (не должна быть отрицательной).

    • По умолчанию: 0.

    • Применение: к grid-контейнерам.

    • Наследование: нет.

    Сокращенное свойство grid-gap для указания ширины пространства для строк и столбцов за один прием, значения grid-row-gap, grid-column.

  • Выравнивание отдельных элементов justify-self, align-self.

    • Значения: start|end|center|left|right|self-start|self-end|stretch|normal|auto (для обоих свойств).

    • По умолчанию: auto (в соответствии со значением justify-items или align-items соответственно).

    • Применение: к grid-элементам.

    • Наследование: нет.

    • Значения: start|end|center|left|right|self-start|self-end|stretch|normal (для обоих свойств).

    • По умолчанию: normal (stretch для незаменяемых элементов, start - для заменяемых).

    • Применение: к grid-контейнерам.

    • Наследование: нет.

  • Выравнивание дорожек в grid-контейнере justify-content, align-content.

    • Значения: start|end|center|left|right|stretch|space-around|space-between|space-evenly (для обоих свойств).

    • По умолчанию: start.

    • Применение: к grid-контейнерам.

    • Наследование: нет.

Адаптивный дизайн

Методика адаптивного веб-дизайна

Основные компоненты адаптивного веб-дизайна:

  • гибкую сетку - возможность сжиматься и перетекать в доступное пространство браузера;

  • гибкие изображения - изображения и медиаобъекты имеют возможность масштабироваться;

  • медиазапросы CSS - возможность досталять наборы правил только на те устройства, которые соответствуют определенным критериям, таким как ширина и ориентация;

  • как дополнение элемент области просмотра meta- позволяет ширине веб-страницы соответствовать ширине экрана.

Организовывать адаптивность необходимо с построения наименьшего фрагмента контента, одноколоночной страницы.

Оптимизация для широкого диапазона области просмотра

  • Общие понятия .

  • Иерархия контента

    - выбор контент-стратегии позволяет определить необходимые точки прерывания, для маленьких областей просмотра рекомендуется использовать условную загрузку с использованием JS.

  • Макет

    - дизайнеры начинают с одноколоночного макета, перекомпановывая элементы в более широкое число колонок, когда есть место. Оптимальной длиной строки является длина от 45 до 75 символов, включая пробелы. Шаблоны для обработки макетов адаптивными сайтами (Люк Вроблевски "Mobile First"):

    • Сильно текучий - одноколоночный макет для небольших экранов и текучие макеты, охватывающие средние и большие экраны. с ограничением максимальной ширины для предотвращения их слишком большого растяжения.

    • Выпадение столбца - переключение между макетами с одним, двумя или тремя столбцами в зависимости от доступного пространства.

    • Сдвиг макета - собственный макет для различных сред.

    • Небольшая настройка - одноколоночный макет с настройкой типа, интервалов, изображений для работы с устройствами с различными размерами.

    • Вне холста - компоненты расположены слева и справа от экрана и отображаются по запросу.

  • Типографика, примеры параметров контента для типографики:

    • Гарнитура шрифта - для маленьких экранов в приоритете разборчивость, для больших - стиль, для маленьких размеров устройств исключить шрифты с засечками.

    • Размер шрифта - возможность использования медиазапросов на основе em.

    • Высота строк - line-height должен быть около 1.5, на маленьких экранах от 1.2 до 1.5, для больших от 1.4 до 1.6.

    • Поля - для маленьких экранов левые и правые поля минимальны (2-4%) в главном столбце.

    • Вне холста - компоненты расположены слева и справа от экрана и отображаются по запросу.

  • Навигация, некоторые шаблоны:

    • Панель навигации в верхней части шаблона - одна или две строки в верхней части экрана (для навигации пальцами ссылка должна иметь размер не менее 44px).

    • приоритет + - наиболее важные навигационные ссылки в верхней части экрана рядом со ссылкой more.

    • Форма ввода select - использование пользовательского интерфейса выбора в виде прокручиваемого списка.

    • Связь с меню в колонтитуле - ссылка Menu вверхней части страницы и связана с расположенной в нижней части страницы полной системой навигации.

    • Аккордионная субнавигация - используется при наличии вложенных подменю.

    • Переключатели сдвигающие и наложения - навигационное меню скрыто, но раскрывается вниз при нажатии на значок меню, при этом контент сдвигается вниз (push-переключатели) или меню накладывается на контент (overlay-переключатели).

    • Вне холста - размещение навигационных ссылок на панели, находящейся за пределами экрана слева или справа от основного содержимого страницы и отображающейся при нажатии на значок меню.

  • Изображения, ключевые моменты:

    • Адаптивные методы разметки изображений - предоставление несколько версий ключевых изображений.

    • Использование наименьшей версии изображения - уменьшение загрузки данных.

    • Отображение важных деталей на любых устройствах - использование обрезки изображения для семантики на маленьких экранах.

    • исключить размещение текста в графике - при невозможности, для маленьких экранов необходима версия с увеличенным текстом.

  • Специальный контент, к нему относится:

    • Формы - для придания гибкости решением является использование Flexbox.

    • Таблицы - сложный объект для адаптации, имеет три основных подхода: прокрутка, укладка (узкий свиток с повторением заголовка) и сокрытие (пользователь выбирает столбцы, которые ему необходимы).

    • Интерактивные элементы - сложный объект для адаптации, возможно решение стоит преоставить браузеру и его приложениям.

Медиазапросы

  • Структура медиазапроса "@media type and (feature:value){стили для браузеров}", например @media screen and (orientation:landscape){body{background:blue;}}.

  • Наиболее часто используются следующие мультимедийные типы: all, print, screen, speech (речь)

  • С помощью медиазапросов можно проверять следующие медиасвойства:width, height, orientation (portrait или landscape), aspect-ratio (отношение ширины области просмотра к высоте), color (битовая глубина экрана), color-index (количество цветов в таблице соответствия цветов), monochrome (количество битов на пиксел в мннохромном устройстве), resolution (плотность пикселов на устройстве), scan (применяет ли тип медиа tv прогрессивную или чересстрочную развертку), grid (использует ли устройство строчный дисплей).

  • В отдельных случаях для разных устройств возможно посместить стили в отдельные файлы, для загрузки необходимого файла таблицы стилей используется атрибут media в элементе link, например link rel=""stylesheet" href="/" media="screen and (min-width:1024px)". Кроме этого, выполнять медиазапросы можно с помощью правил @import без использования ключевого слова media - @import url("/") screen and (min-width:124px);.
