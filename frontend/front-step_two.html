СЕМАНТИЧЕСКИЕ ЭЛЕМЕНТЫ

СТРУКТУРА

Основные элементы структуры веб-страницы

  • Объявление типа документа (< !doctype html >) - документ создан на HTML5.
  • Элемент < html > — он включает все элементы в документе. Внутри элемента < html >:
    1. Элемент < head > - заголовок, таблицы стилей, сценарии и метаданные.
      • элемент < meta > содержат информацию о документе.
      • элемент < title > - описательный заголовок.
    2. Элемент < body > содержит контент.
        3. Элемент < header >...< /header>. Верхний колонтитул.Используется для разметки вводного материала, как правило, логотип, заголовок, элементы навигации. Элементы навигации помещаются в элемент < nav >...< /nav >. Элемент < header > также может включать заголовок статьи
      • Тематические разрывы (горизонтальные линейки) < hr >.Тематические разрывы,логические разделители, показываеют, что одна тема завершена и начинается следующая.
      • Элемент < main >...< /main>. Основная область контента.В этом разделе находится уникальный для данной страницы контент.
      • Элемент < footer >...< /footer >.Может применяться ко всему документу или разделу (статье), если в этом есть семантический смысл. Как правило содержит информацию об авторе, связанных документах, элементах навигации.
      • Элемент < section >...< /section >. Разделы. Тематическое группирование контента.Состоит из заголовка и контент сгруппированный по смыслу.
      • Элемент < article >...< /article >. Статьи.Автономная многоразовая композиция.Используется для разметки отдельных подразделов.Если текст автономный, то его лучше помещать в < articale >
      • Элемент < nav >...< /nav >. Основные навигационные ссылки.
      • Элемент < aside >...< /aside >. Частично связанный с контентом материал.
      • Элемент < div >...< /div >
      • Элемент < address >...< /address >. Формирование контактной информации. Используется обычно в footer для разметки контактной информации автора.

Структурирование данных

Популярные стандарты структурирования данных:

  • Microformats

    Является широко применяемым соглашением, например стандарт определяет значения атрибутов id, class, rel для определения людей, организаций, событий, продуктов и др.

  • Microdata

    Является HTML-стандартом WHATWG (Web Hypertext Application Technology Working Group, Рабочая группа по технологиям веб-гипертекста), который применяет специфические атрибуты: item-scope, itemtype, itemprop, itemid, itemref. Данный стандарт в 2013 году исключен из HTML5

  • RDFa

    Является HTML-стандартом (Resource Description Framework in Attributes, Фреймворк описания ресурсов в атрибутах), который применяет специфические атрибуты: vocab, typeof, property, resourse, prefix.

  • RDFa Lite

    Является упрощенной версией RDFa.

  • JSON-LD

    Другой подход в части размещения типов объектов и их свойств в удаленном из разметки сценарии (JavaScript Object Notation to serialize Linked Data, запись объекта JavaScript, применяемая для сериализации связанных данных).

Улучшение доступности документа с помощью ARIA(Accessible Rich Internet Application, Доступные многофункциональные интернет-приложения)

ARIA - стандартизированный набор атрибутов, использование которых упрощает навигацию по страницам и применение интерактивных функций.Определяет роли, состояния и свойства, которые могут добавлять в разметку и сценарии для поддержки более насыщенной семантической информации Поддерживается Рабочей группой инициативы доступности веб-сайтов (WAI).

Роли - описывают функции или цели элемента в контексте документа.Являются подсказкой для ассистивных технологиях. К ролям относятся: alert, button, dialog, slider, menubar, navigation, banner, contentinfo, complementary, main.Пример применения - < ul id="table" role="toolbar" >

Состояния и свойства обозначаются атрибутами с префиксом aria-, например: aria-disabled, aria-describedby

Текстовые и обобщающие элементы HTML

Группировка элементов контента и атрибутов.

  1. Абзацы < p >...< /p > отображаются на новых строках с заданными промежутками между ними. Абзацы могут содержать текст, изображения и другие строчные элементы (фразовый контент).

  2. Длинные цитаты
    < blockquote >...< /blockquote >    Длинная цитата, цитирование на уровне блока.Рекомендуется в контент между указанными тегами включать такие элементы, как абзацы, заголовки, списки. Существует элемент < q >, который применяется для разметки коротких цитат.

  3. Тег < cite >...< /cite > - цитата, ссылка на заголовок работы.Ссылка на пример.
  4. Элементы div и span
    < div class="#">...< /div >    Обобщающий элемент блочного уровня. < span >...< /span > Обобщающий элемент строчного уровня.
    Обобщающие элементы используются для группировки элементов в единый контейнер. Атрибуты id и class используются при структурировании контента и могут применяться одновременно. Кроме этого эти атрибуты являются глобальными атрибутами и могут применяться со всеми элементами HTML.

    Атрибут id служит для придания элементу в документе уникального идентификатора, должно использоавться в тексте только один раз.

    Атрибут class собирает элементы в концептуальные группы, может использоваться несколькими элементами одновременно.

  5. Фигуры
    < figure >...< /figure >    Связанное изображение или ресурс. < figcaption >...< /figcaption > текстовое описание фигуры. Элемент определяет контент, который иллюстрирует некоторый элемент в тексте.

  6. Предварительно отформатированный текст
    < pre >...< /pre >    Предварительно отфоратированный текст используется для контента с семантически значимыми пробелами. Для сохранения пробелов и символов разрыва строки исеользуется свойство CSS white-space:pre).

  7. Тег < hr >...< /hr > - тематический разрыв на уровне абзаца (горизонтальная линейка).

Элементы и атрибуты списка.

Неупорядоченные списки.
Наборы пунктов, отображаемые в произвольном порядке. Теги < ul >...< /ul >, элемент списка в неупорядоченном списке < li >...< /li >. Пункты списка, отображаются с маркером перед каждым элементом списка. Вид маркеров можно менять с помощью CSS. В неупорядоченный список нельзя помещать другие элементы или текст без тегов, но в элемент списка можно поместить любой тип элемента.
Упорядоченные списки.
Cписки, для которых важна последовательность пунктов.Теги < ol >...< /ol >, элемент списка в упорядоченном списке < li >...< /li >. Отображаются с порядковым номером перед каждым элементом списка. Имеет атрибут value - предоставляет значения для li в ol.Отображение списка можно менять с помощью CSS. С помощью атрибута start элемента < ol > можно указать начальный номер - start="5". Можно изменить нумерацию на обратную с помощью reversed.
Списки описаний.
Cписки, которые предствлены парами "имя-значение".
Теги < dl >...< /dl >, список определений, название, термин (метка) < dt >...< /dt >, термин, значение (описание, определение) < dd >....В элементы dt нельзя помещать заголовки или элементы группировки, например, абзацы. Элементы < dd > - определения, могут включать в себя любой тип контента.Элементов < dd > может быть несколько.

ТЕГИ И АТРИБУТЫ

Теги - специальные средства языка HTML, которые определяют структуру и внешний вид документа при его отображении браузером.

Что нужно знать о тегах: 1) теги бывают одиночными, например < hr > - разделитель. 2) теги бывают парными, например < p > < /p > - абзац. 3) теги бывают вложенными, при этом должен соблюдаться порядок закрытия тегов. 4) теги имеют атрибуты.

Атрибуты - это инструкции, которые уточняют или модифицируют элемент.
Синтаксис для атрибута: < элемент > название_атрибута1="значение" название_атрибута2="значение">.

Что нужно знать об атрибутах: 1) атрибуты располагаются после имени элемента и только в открывающем теге, но никак не в закрывающем; 2) к элементу можно применять несколько атрибутов, разделенных пробелами в открывающем теге; 3) большинство атрибутов принимают те значения, которые указаны после знака равенства.Некоторые значения атрибутов описываются одиночными словами,например, атрибут checked, эквивалентен указанию check="checked".

Теги и атрибуты для ссылок

Тег < a href="/" > < img src="#.gif" alt="какое-то название"> < /a > - якорь (привязка) или гипертекстовая связь с ссылкой на изображение. В атрибуте href используются абсолютные и относительные ссылки. Абсолютные ссылки - это ссылки, которые представляют собой полный интернет адрес файла и начинаются с http://www.xxxxxxx.com/". Более сложными являются относительные ссылки - описывают путь относительно текущего документа xxxxxx/yyyyy/zzzzz.html, особенностью формирования относительного пути является использование соглашения "точка-точка-слэш" ../ для перехода на один уровень каталога вверх. Относительная ссылка может формироваться от корневого каталога посредством указания в начале пути слэша /xxxxxx/yyyyy/zzzzz.html без указания имени корневого каталога (целесообразен для любого документа на сервере).
Для страниц с прокруткой актуально применение ссылок на целевой фрагмент страницы. Для этого необходимо обозначить целевой фрагмент с помощью идентификатора фрагмента - атрибута id, а при создании ссылки с помощью символа решетки # обозначить имя фрагмента следующим образом href="zzzzzzz.html#fragment.При этом этот способ работает и для внешних ссылок на другие сайты, конечно при наличии в коде этих сайтов соответствующих идентификаторов.
Для отображения ссылки в новом окне с помощью атрибута тега < a > target="_blank", значение атрибута может иметь любое имя, но если имя начинается с нижнего подчеркивания, то ссылка открывается в новой вкладке браузера.

Существуют специальные виды ссылок, например:

  • почтовые ссылки mailto, их необходимо указываеть в атрибуте href следующим образом < a href="mailto:XXXXXX@YYYYYY.com" >;
  • телефонные ссылки tel — < a href="tel:+7-800-800-800-8888">;

Теги и атрибуты для работы с изображениями

Тег < img src="/xxxxx.png alt="">...< /img > - добавление изображений. Форматы растровых изображений: png, jpeg, gif, WebP, jpeg-xp. Формат векторного изобюражения - svg. Атирибуты src и alt являются обязательными, значением атрибута src является ссылка на загружаемое изображение, аналогично href, значением атрибута alt является альтернативный текст, который отображается если изображение не загрузилось, используется screenreader, поисковыми роботами.

Поддержка изменения размеров осуществляется с помощью атрибутов width="ширина изображения в пикселях" и height="высота изображения в пикселях". При указании одного размера изображение масштабируется пропорционально. При адаптивном подходе к верстке использование указанных атрибутов недопустимо, их целесообразно использовать для логотипов и пиктограмм. Ссылка на пример.
У элемента img существует атрибут usemap, который позволяет создавать карту изображения на стороне клиента.

Добавление векторных изображений.
Scalable Vextor Graphics - масштабируемая векторная графика.Формы и контуры изображений SVG описаны математически на стандартизированном языке разметки, который является приложением языка XML (Extensible Narkup Language, расширяемого языка разметки).
Cпособы вставки SVG-изображений:

  • с помощью тега < img >, но этот вариант подходит только для статических изображений, так как как обернутых таким образом изображениям невозможно применять стили, скрипты и др.
  • копирование контента SVG-файла и вставка его а HTML-документ < svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 300 180"> < rect width="300" height="180" ﬁ ll="purple" rx="20" ry="20"/> < text x="40" y="114" ﬁ ll="yellow" font-family="'Verdana-Bold'" ontsize=" 72"> hello! < /text> < /svg>, недостатком указанного способа является добавление существенного блока кода в документ.
  • размещение SVG-изображений является использование универсального для мультимедийных объектов элемента object, который определяет тип мультимедиа (изображение svg+xml) и указывает на файл с помощью атрибутов type и date соответственно,< object type="image/svg+xml" data="pizza.svg"> < div style="background-image: url(pizza.png); width:100px; height:100px;" role="img" aria-label="pizza">< /object>. Недостатком данного способа является невозможность применения таблицы стилей к изображению.Ссылка на пример.SVG-изображения оптимально использовать в качестве фонового изображения CSS.
  • использование элемента < picture >, как правило этот элемент используется для создания нескольких версий изображений с использованием различных форматов, < picture > < source type="image/svg+xml" srcset="pizza.svg"> < img srcset="pizza.png" alt="No SVG support">< /picture>. Принцип размещения аналогичен предыдущему способу.

Обработка адаптивных изображений.

Основной принцип - право браузера выбирать наиболее соответствующее изображение из набора на основе сведений о текущей среде для просмотра.
Сценарии обработки адаптивного изображения:

  • поддержка очень больших изображений для экранов с высоким разрешением (плотностью) осуществляется с помощью атрибута элемента < img >srcset, который включает список параметров изображения, выбираемых браузером. Каждый элемент в списке состоит из URL изображения и x-дескриптора (соотношения логических и физических пикселов), который определяет целевое соотношение пикселов на устройстве, при этом в элементе < img > остается элемент src для укзания значения изображения по умолчанию.Пример разметки:
    < img
    src="/images/turkey-200px.jpg" alt=""
    srcset="/images/turkey-400px.jpg 2x,
    /images/turkey-600px.jpg 3x" >
  • поддержка набора изображений различных размеров, отображаемых на экранах с разными размерами, осуществляется с использованием атрибутов элемента < img > srcset и sizes.В отличие от предыдущего сценария атрибут srсset содержит w-дескриптор (дескриптор ширины, поддерживающий фактическую пиксельную ширину для каждого изображения). Использование w-десмкриптора целесообразно если изображения отличаются только масштабом, при этом необходимо применять атрибут sizes, который сообщит браузеру приблизительный размер страницы.Пример разметки:
    < img src="strawberries-640.jpg" alt="baskets of ripe strawberries"
    srcset="strawberries-480.jpg 480w,strawberries-960.jpg 960w,strawberries-1280.jpg 1280w, strawberries-2400.jpg 2400w"
    sizes="100vw"> (vw-единица ширины области просмотра).
    При использовании различных макетов атрибут sizes состоит из двух частей: медиаусловия (media-feature), описывающее в качестве параметра condition ширину области, и размер length, определяющий ширину, которую изображение займет в макете при выполнении медиаусловия. Пример разметки:
    sizes="(max-width:480px)100vw,(max-width: 960px) 70vw, 240px">
  • поддержка изображений с различной степенью детализации (принципы художественного оформления, art direction, возможность удаления неглавных деталей на небольших экранах) осуществляется с использованием элемента< picture >. Данный элемент не имеет атрибутов и представляет собой оболочку для некоторого числа элементов source и элемента img. Элемент img является обязательным и размещается последним в наборе. Пример разметки:
    < picture >
    < source media="(min-width: 1024px)"
    srcset="icecream-large.jpg">
    < img src="icecream-small.jpg" alt="Lorem ipsum dolor">
    < /picture >
    Кроме этого в элементе < picture > может содержаться атрибут sizes.
    Атрибут media поддерживает медиазапрос для проверки текущих условий просмотра.
  • поддержка альтернативных форматов изображений, которые сохраняют одинаковые изображения при использовании файлов гораздо меньших размеров, также основана на элеменет picture, в котором используется атрибут type, который указывает тип файла (тип MIME (Multipurpose Internet Mail Extension, многоцелевые расширения почты Интернет). Пример разметки:
    < picture >
    < source type="image/webp"
    srcset="icecream-large.webp">
    < img src="icecream-small.jpg" alt="Lorem ipsum dolor">
    < /picture >

ФОРМЫ И ВАЛИДАЦИЯ

ИНТЕРАКТИВНАЯ ФОРМА

Элемент< form >...< /form > - интерактивная форма предназначенная для обмена данными между пользователем и сервером. Этот элемент не может включать элементы form, то есть не допускается вложенность элементов.

Элемент form содержит атрибуты:

  • атрибут action - определяет местоположение приложения или сценария, который используется для обработки формы. Пример разметки: < form action="/XXXXX.php" method="POST">...< /form >
  • атрибут method - указывает, каким образом информация должна направляться серверу. Существуют два метода, применяемых для отправки закодированных данных на сервер:
    • Метод GET - данные закодированной формы прикреаляются прямо к URL, направляемому серверу, при этом непосредственно URL отделяется от данных знаком ?. Этот метод используют в формах когда необходимо сохранить результаты поиска, список товаров, а также для перехода по гиперссылке с URL.
      Метод не подходит, если при отправке формы выполняются действия по удалению или доабвлению данных в базу данных, обработке личной или финансовой информации, а также если отправляется большое количество данных или осуществляется загрузка файлов.
    • Метод POST - браузер направляет серверу отдельный запрос, содержащий специальные заголовки, за которыми и следуют данные.
    • Атрибут accept-charset - устанавливает кодировку, в которой сервер может принимать и обрабатывать данные. Пример разметки: < form action="/XXXXX.php" method="POST" accept-charset="UTF-8">...< /form >
    • Атрибут autocomplete - включает автозаполнение формы. Пример разметки: < form action="/XXXXX.php" method="POST" accept-charset="UTF-8" autocomplete="on|off">...< /form >
    • Атрибут enctype - определяет способ кодирования данных формы. Пример разметки: < form action="/XXXXX.php" method="POST" accept-charset="UTF-8" autocomplete="on|off" enctype="application/x-www-form-urlencoded | multipart/form-data | text/plain>...< /form > . При использовании поля загрузки файла обязательно использование значения multipart/form-data.
    • Атрибут novalidate - отменяет встроенную проверку данных формы на корректоность ввода. Пример разметки: < form action="/XXXXX.php" method="POST" accept-charset="UTF-8" novalidate>...< /form > . Проверка осуществляется браузером автоматически и происходит для полей email, url и при наличии атрибутовpattern, required.
    • Атрибут target - имя окна или фрейма, куда обработчик будет загружать возвращаемый результат. Пример разметки: < form action="/XXXXX.php" method="POST" accept-charset="UTF-8" autocomplete="on|off" target="имя_окна name|_blank|_self|_parent|_top">...< /form >.

ЭЛЕМЕНТЫ УПРАВЛЕНИЯ ФОРМОЙ

Глобальные атрибуты для элементов форм:

Атрибут name - все элементы управления формой, которые осуществляют сбор данных и их отправку на сервер, должны включать атрибут name.
Элементы управления формой не могут именоваться произвольным образом, так как это имя использует веб-приложение (сценарий), обрабатывающий данные.Элементы управления формой не могут именоваться произвольным образом, так как это имя использует веб-приложение (сценарий), обрабатывающий данные.
Атрибут disabled - не разрешает пользователям взаимодействовать с элементом управления формой.При его использовании, элемент формы нельзя выбрать, он отображается серым цветом. Отключенное состояние может изменяться только с помощью сценария. Указанный атрибут используют для ограничения доступа к некоторым полям формы с учетом введенных данных и может применяться с любым элементом формы или элементом fieldset.Поле disabled не передается при отправке формы.
Атрибут readonly - не разрешает пользователям изменять значение элемента управления формой, хотя его можно выбрать. При его использовании, установка значений может осуществляться посредством сценариев. Указанный атрибут может применяться с элементами textarea и с основанными на тексте элементами управления вводом. Поле readonly передается при отправке формы.

Разметка элементов управления формой.

Элементы управления вводом текста.
Если форма имеет поле для ввода текста следует применять защищенный протокол HTTPS.

  • Однострочное текстовое поле.
    Пример разметки: < label >Цвет неба< input type="text" name="" id=" " value="Синий" maxlength="50" > - поле ввода одной строки текста. Доступность поля формы улучшена применением элемента label.

    Атрибуты элемента input :

    • type - основной атрибут применяется для определения вида элемента input (текстовое поле (text), поле с паролем (password), переключатель (radio), флажок (checkbox), скрытое поле (hidden), кнопка (button), кнопка для отправки формы (submit), кнопка для очистки формы (reset), поле для отправки файла (file) и кнопка с изображением (image));
    • name - атрибут применяется для указания имени переменной;
    • value - атрибут определяет заданный по умолчанию текст, который при загрузке формы отображается в поле. При сбросе формы она возвращается к этому значению. Альтернативой является атрибут placeholder, который содержит подсказку, что именно вводить в поле, при этом значение placeholder не направляется на сервер вместе с формой.
    • maxlength, minlegth - атрибуты определяющие максимально и минимально допустимое количество символов, необходимое для программы обработки формы.
    • size - атрибут, определяющий длину поля ввода, по умолчанию 20 символов (обычно задается CSS).
    • accept - атрибут, устанавливающий фильтр на типы файлов, которые можно отправить через поле загрузки файлов, тип файла указывается как MIME-тип, в HTML5 допускается указывать audio/*, video/*, image/*.
    • accesskey - переход к полю с помощью сочетания клавиш, в значении атрибута указывается цифра 0-9 или латинская буква a-z. Для каждого браузера используются различные сочетания клавиш.
    • align - определяет выравнивание изображения. Для каждого браузера используются различные сочетания клавиш.

  • Многострочное поле для ввода текста.
    Пример разметки: < label >Особенности атмосферы < textarea name=" " id=" " rows="5" cols="50" wrap="soft" value="Синий" maxlength="50" minlegth=10 placeholder="Много написано">

    Атрибуты элемента textarea:

    • rows - атрибут применяется для определения количества строк, которые отображает текстовая область;
    • cols - атрибут применяется для определения ширины, представленной количеством символов (обычно используется CSS), в HTML5 по умолчанию равен 20.
    • wrap - атрибут определяет, сохраняются ли при отправке формы мягкие разрывы (значение soft - не сохраняет разрывы строк, hard- сохраняет разрывы строк.
    • autofocus - логический атрибут. Если указан, при загрузке документа фокус будет переведен на текстовую область.
    • form - значением атрибута является id формы, к которой относится текстовая область, если элемент находится вне кода формы.
    • required - логический атрибут. Если указан, текстовая область будет определена как обязательная для заполнения, форма не будет отправлена, пока текстовая область не будет заполнена.
    • tabindex - определеяет последовательность перехода между элементами при нажатии на кнопку Tab, принимает значения 0-9.

    В элементе также применяются атрибуты maxlength, minlegth, placeholder,accesskey, а также глобальные атрибуты.

Окно в окне:элемент IFRAME.

Элемент < iframe >...< /iframe > (inlaine frame) - контейнер, который позволяет встроить в документ отдельный HTML-документ или другой веб-ресурс.
При добавлении элемента на страницу на ней создается вложенный контекст просмотра, в котором отображается внешний ресурс.
К атрибутам элемента относятся:
  • Атрибут name - имя фрейма.
  • Атрибут src - путь к файлу, содержимое которого загружается во фрейм, возможно указывать имя функции JavaScript.
  • Атрибут srcdoc="< html-код >" - хранит содержимое фрейма непосредстванно в атрибуте.
  • Атрибуты width и height - ширина и высота фрейма, может задаваться в процентах от размеров родительского элемента.
  • Атрибут align="absmiddle | baseline | bottom | left | middle | right | texttop | top" - определяет как фрейм будет выравниваться по краю, а также способ обтекания текстом.
  • Атрибут allowtransparency - устанавливает прозрачный фон фрейма, через который виден фон страницы.
  • Атрибут frameborder="yes | no |0 | 1" - устанавливает, отображать границу вокруг фрейма или нет.
  • Атрибуты hspace и vspace - горизонтальный и вертикальный отступы от фрейма до окружающего контента в пикселях.
  • Атрибуты marginheight и marginwidth - отступы сверху, снизу, слева, справа до границы фрейма.
  • Атрибут sandbox="allow-same-origin |allow-top-navigation |allow-forms |allow-scripts" - позволяет задать ограничения на контент загружаемый во фрейме: разрешает загружать содержимое фрейма, воспринимая его из того же источника, что и родительский документ, блокирует всплывающие окна; позволяет открывать ссылки фрейма в родительском документе; позволяет содержимому фрейма отправлять формы; разрешает запуск и выполнение скриптов. Создание всплывающих окон запрещено.
  • Атрибут scrolling="auto | no | yes" - способ отображения полосы прокрутки во фрейме.
  • Атрибут seamless - определяет, что содержимое фрейма должно отображаться так, словно оно является частью документа.

ВИДЕО- И АУДИОПЛЕЕРЫ

Поддержка видео-аудиоплееров основана на использовании интерфейса программирования приложений API.
API (Application Programming Interface) - это стандартизированный набор команд, имен данных, свойств, действий и т.д., позволяющий приложениям взаимодействовать друг с другом.

Видеоплеер VIDEO.

Элемент < video >...< /video > - добавляет на страницу видеоплеер.
К атрибутам элемента относятся:
  • Атрибут src - определяет адрес ресурса.
  • Атрибут widh и height - определяют "пиксельный" размер области, которую занимает встроенный медиаплеер.
  • Атрибут crossorigin - определяет каким образом элемент обрабатывает запросы, поступающие от других источников (серверов). Имеет значения anonymous|use-credentials.
  • Атрибут poster - url-ссылка на изображение, отображаемого на месте видео перед его воспроизведением.
  • Атрибут controls - отображение встроенных элементов управления мультимедийными объектами. Можно создать собственный интерфейс с помощью CSS и JavaScript.
  • Атрибут autoplay - автоматическое воспроизведение видео после загрузки достаточной для воспроизведения без остановки части файла.
  • Атрибут loop - повторное воспроизведение видео.
  • Атрибут muted - воспроизведение видео без звука.
  • Атрибут preload - управление извлечением видеоданных после загрузки страницы.Значения auto|none|metadata показывают сколько потребуется буферизации мультимедийного ресурса.

Для выбора нужного для браузера видеофайла можно использовать элементы < source >, что позволит обеспечить поддержку различными браузерами.

Элемент имеет те же атрибуты, что и элемент < video >, за исключением width, height, poster.

Добавление текстовых дорожек.

Элемент < track >...< /track > - добавление к встроенному медиаобъекту синхронизированного текста.

К атрибутам элемента относятся:
  • Атрибут src - указывает текстовый файл.
  • Атрибут kind - указывает тип предоставляемой текстовой аннотации subtitles, caption, descriptions, chapters, metadata. Если значение атрибута subtitles, то добавив атрибут srclang, с помощью двухбуквенного языкового тега IANA указав язык.
  • Атрибут label - предоставляет для дорожки имя, которое можно применить в интерфейсе для выбора конкретной дорожки.
  • Атрибут default - помечает конкретную дорожку как заданную по умолчанию (может применяться только на одной дорожке в элементе).
  • Атрибут loop - повторное воспроизведение видео.
  • Атрибут muted - воспроизведение видео без звука.
  • Атрибут preload - управление извлечением видеоданных после загрузки страницы.
